# Plan: 01-03: Session & Logger Refactor

Modernize the `Session` and `Logger` core classes by converting them to instance-based logic managed by the Service Container.

## Proposed Changes

### Core

#### [MODIFY] [Session.php](file:///c:/Users/TESS%20LARON/Desktop/REVISED/src/Core/Session.php)
- Convert all static methods (`start`, `get`, `put`, etc.) to instance methods.
- Implement a static Bridge for backward compatibility (similar to Database).

#### [MODIFY] [Logger.php](file:///c:/Users/TESS%20LARON/Desktop/REVISED/src/Core/Logger.php)
- Remove the static `instance` property and static `instance()` method.
- Convert `debug`, `info`, `warning`, `error` to instance methods that proxy to the internal Monolog instance.
- Update the constructor to initialize the Monolog handler.
- Register as a Singleton in the Container.

#### [MODIFY] [app.php](file:///c:/Users/TESS%20LARON/Desktop/REVISED/config/app.php)
- Update the bootstrap sequence to register `Logger`, `Database`, and `Session` into the Container.
